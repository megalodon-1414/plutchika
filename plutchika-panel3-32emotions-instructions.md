## 1. 組み合わせロジック（重要・そのまま実装してください）

プルチック環は8つの基本感情が円環状（1→2→3→4→5→6→7→8→1…）に並んでいます。
選んだ2つの感情ID（1〜8）の**円環上の距離**によって、組み合わせ感情のtierが決まります。

```js
function circularDistance(a, b) {
  const diff = Math.abs(a - b);
  return Math.min(diff, 8 - diff);
}
```

- distance === 0（同じ感情を選択）→ ピュア感情（basic8）をそのまま表示
- distance === 1 → primary（8種）
- distance === 2 → secondary（8種）
- distance === 3 → tertiary（8種）
- distance === 4 → 対極の感情同士（例：喜び⇔悲しみ）。**組み合わせ感情は存在しない**ので、「対極の感情です」等のメッセージを表示するか、そもそも選択できないようにする（実装しやすい方でOKですが、どちらにするかは確認事項として一度ユーザーに聞いてください）

function getCombinedEmotion(idA, idB) {
  if (idA === idB) {
    return { ...basic8.find(e => e.id === idA), tier: "basic8" };
  }
  const [a, b] = [idA, idB].sort((x, y) => x - y);

## 3. 色の混色ロジック

- 2つの感情が異なる場合、表示する文字色は「2つの基本感情の色を混ぜた色」にしてください
- HSL空間で色相(hue)を混ぜると虹色の中間色として自然に見えるはずなので、RGB平均よりHSL平均を推奨します（実装しやすい方で構いませんが、見た目を確認して不自然なら調整してください）
- 同じ感情同士（ピュア感情）の場合は、混色せずそのまま元のbasic8の色を使う

```js
// 参考実装イメージ（既存のカラー形式に合わせて調整してください）
function blendColors(hslA, hslB) {
  // hue は円環なので、単純平均だと逆側に振れることがある点に注意
  // （最短経路で平均するロジックを推奨）
  const h = averageHueShortestPath(hslA.h, hslB.h);
  const s = (hslA.s + hslB.s) / 2;
  const l = (hslA.l + hslB.l) / 2;
  return { h, s, l };
}
```

## 4. UI挙動

1. 花Aの花びらを1枚クリック → 選択状態（ハイライト）になる。もう一度同じ花びらをクリックすると選択解除
2. 花Bも同様に1枚選択可能
3. 花A・花Bどちらも選択されている状態になった瞬間、2つの花の下・中央に組み合わせ感情の文字を表示（フェードインなどの軽い演出でOK）
4. 花Aまたは花Bの選択が解除されたら、表示中の感情名も消える
5. 花びらは常に1つの花につき1枚のみ選択可能（同じ花で別の花びらを選ぶと選択が切り替わる）
