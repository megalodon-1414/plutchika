# PLUTCHIKA 深掘りルート ページ遷移 実装指示（Claude Code 引き継ぎ用）

このファイルは `plutchika-intro-handoff-v2.md` の続きの作業指示です。intro-redesign ブランチで作業中の Claude Code セッションに、このファイルの内容をそのまま貼るか読み込ませてください。

## 0. 前提・背景

必須ルート（ロゴ→画面1・ようこそ→画面2・プルチック環→画面3・搭乗）は既存の仕様どおり実装済み／実装中です。今回追加するのは、必須ルートとは別の「深掘りルート」というページで、任意で読める補足コンテンツです。

深掘りルートの内容は、以下の5つのテーマを順番に並べたものです（本文はまだ仮のプレースホルダーで構いません。最終テキストは別途詰めます）。

1. 感情ラベリングの重要性（心理学・扁桃体の話）
2. 感情粒度・語彙の解像度の話（Barrett）
3. プルチック環の詳しい成り立ち（8感情の分類・複合感情の理論的背景）
4. なぜ検索でなく探索なのか（セレンディピティ）
5. なぜこの没入感なのか（フロー理論・注意回復理論）

## 1. 今回やってほしいこと

1. 深掘りルート用の新しいページ（ルート）を1つ作る
2. そのページの中に、上記1〜5のパネルを縦に5つ並べる（スクロールで全部読める構成）
3. 必須ルートの画面1・画面2の本文中の特定の単語をリンクにして、深掘りページの該当パネルへ飛べるようにする
4. 深掘りページから必須ルートへ戻れる導線をつける

## 2. 深掘りページの構成

- ルート名は仮で `/deep-dive` としますが、既存の命名規則があればそちらに合わせてください
- 各パネルには一意のアンカーIDをつける：`panel-1` 〜 `panel-5`（上記1〜5に対応）
- 表示順は必ず 1→2→3→4→5 の順（必須ルートの論理展開と対応しているため、順番を変えないでください）
- 各パネルの中身は「見出し」＋「本文」のシンプルな構成でOKです（必須ルートのようなフック／サブコピーの多層構造は不要）
- **見た目のトーン**（フォント、文字色、余白、パネルの背景処理など）は必須ルートで使っているテキストパネルのスタイルを流用してください
- **重要な確認事項**：これは通常の縦スクロールページとして実装する想定で、必須ルートにある「球体が回転してテキストが浮かび上がる」3D演出は流用しない想定です。もしこの想定が違う（3D演出ごと深掘りページにも使いたい）場合は、実装前にユーザーに確認してください

パネルの仮本文（プレースホルダーとして使ってください。最終稿ではありません）：

```
panel-1（ラベリングの重要性）
仮見出し：名づけると、なぜ心は落ち着くのか
仮本文：心理学には「感情ラベリング」という考え方があります。悲しいとき、怒っているとき、その気持ちに具体的な名前をつけるだけで、扁桃体という脳の警報装置のような部分の反応が静まることが、脳科学の研究で示されています。

panel-2（感情粒度）
仮見出し：語彙がないと、感情は"見えない"
仮本文：心理学者リサ・フェルドマン・バレットは、感情を細かく言い分けられる能力を「感情粒度」と呼びました。感情は、名前という道具がなければ輪郭を結べません。

panel-3（プルチック環の詳細）
仮見出し：プルチックはなぜ「8つ」を選んだのか
仮本文：心理学者ロバート・プルチックは、喜び・信頼・恐れ・驚き・悲しみ・嫌悪・怒り・期待という8つを、生存に関わる基本的な反応として位置づけました。隣り合う感情同士が混ざり合うことで「後悔」や「愛」のような複合的な感情が生まれる、という考え方も提示しました。

panel-4（探索の哲学）
仮見出し：探しに行くのではなく、出会いに行く
仮本文：欲しい言葉が最初からわかっているなら、検索窓に打ち込めば済みます。PLUTCHIKAが感情の地図を歩いて回る形にしているのは、探しているつもりのなかった言葉に、たまたま出会ってもらうためです。

panel-5（没入感・フロー理論）
仮見出し：なぜ、ゆっくり歩くような体験にしたのか
仮本文：何かに没頭しているとき、人は時間を忘れ、集中力が高まります。心理学ではこれを「フロー状態」と呼びます。
```

## 3. 必須ルートからのリンク配置（今回確定分）

以下の2箇所は今回確定なので、実装してください。

- **画面1（ようこそ）の本文中**：「心理学」という単語をリンクにし、クリックで深掘りページの `panel-1` へ遷移＋スクロール
- **画面2（プルチック環）の本文中**：
  - 「プルチック環」という単語をリンクにし、クリックで `panel-3` へ遷移＋スクロール
  - 「探索」という単語をリンクにし、クリックで `panel-4` へ遷移＋スクロール

画面2の本文は以下の通り更新してください（誤字「亜笑しました」も修正済みです）：

```
心理学者プルチックが考案した「プルチック環」は、8つの基本感情に色と位置を与え、感情をひとつの地図として表したものです。PLUTCHIKAは、このプルチック環をもとに、感情を表す言葉をひとつひとつ段階に分けて位置づけ、マッピングしました。大きな感情から、少しずつニュアンスの違う言葉へ——地図の上を「探索」するように辿っていくと、今のあなたにぴったりな一語が見つかるはずです。さあ、その言葉を探しに行きましょう。
```

**panel-2・panel-5 はまだリンク元の単語が確定していません。** 深掘りページ自体には両パネルとも用意しておき、通常のスクロールでは読めるようにしておいてください。画面1・画面2からの直接リンクは、本文が確定次第あとで追加します。

## 4. 深掘りページから必須ルートへ戻る導線

- 深掘りページの上部（またはヘッダー）に「必須ルートに戻る」的な戻りリンクを常設してください
- **確認事項**：戻ったとき、ユーザーが元々見ていた画面（画面1 or 画面2）にそのまま戻れるようにしたいです。ロゴ画面からやり直しにならないよう、遷移前の状態（どの画面にいたか）を保持する必要があります。ブラウザの「戻る」で十分なのか、明示的に状態管理が必要なのか、既存のアプリ構造（本体側のルーティング）を踏まえて実装方針を決めてください。判断に迷う場合はユーザーに確認してください

## 5. 実装前に確認してほしいこと（技術判断）

- このリポジトリで React Router 等のルーティングライブラリを既に使っているか。使っていない場合、新規追加するか、それとも状態管理だけで画面切り替えを行うか
- アンカーへのスムーズスクロールは、既存の依存関係にあるものを使うか、CSSの `scroll-behavior: smooth` で十分か
- 深掘りページは `intro-redesign` ブランチのスコープ内で完結させてよいか、本体側のルーティング（`/map` 等）と将来的に統合する前提を考慮する必要があるか

上記の確認事項は、実装を進める前にユーザーに一言確認してから着手してください。
